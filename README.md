## 프로젝트 소개
 
Gen-City는 **generate + city**, **generation + city**의 중의적 의미를 담은 이름입니다.
 
서울 지도에서 부지를 클릭하면 주변 인프라 데이터를 자동 수집하고,  
개발 목표를 입력하면 Gemini AI가 토지이용계획을 추천하는 Flask 기반 웹 애플리케이션입니다.
 
입지 분석에서 투자 수익성까지 연결하는 **end-to-end 부동산 분석 플랫폼**으로, 도시공학적 입지 판단과 금융적 수익성 검토를 하나의 툴에서 수행할 수 있도록 설계했습니다.
 
---
 
## 왜 만들었나

서울에 도착하고 느낄 큰 감정 중 하나는 이질감이라 생각합니다.  
롯데몰, 롯데타워, 롯데월드, 석촌호수 등 거대한 인프라를 가진 잠실역 맞은편에는 아직도 1980년대에 지어진 노후 건물이 그대로 있으며, 서울역 도보권인 후암동은 젠트리피케이션 등 복합적인 문제로 수십 년째 개발도 재정착도 없이 방치되고 있습니다.

이 문제를 해결하려면 더 나은 질문이 필요하다고 생각했습니다.  
그런데 개발 방향성을 판단하는 도구는 없었습니다.  
기존의 부지 분석은 대부분 투자/거래 관점이었습니다.

> "이 땅이 얼마짜리냐"는 답할 수 있어도,  
> "이 땅이 어떤 용도로 개발될 때 적합한가"는 답하지 못했습니다.

Gen-City는 이 두 질문을 하나의 흐름으로 연결합니다.

공시지가, 교통, 생활인프라 접근성 같은 정량적 조건으로 입지를 판단하고, 실거래가 기반 NOI·Cap Rate로 수익성을 검토합니다. 입지 점수가 높은데 Cap Rate도 높다면 저평가된 개발 후보지일 수 있습니다. 그 판단의 근거를 AI가 설명합니다.

## 주요 기능
 
| 기능 | 설명 |
|------|------|
| 🗺️ 지도 클릭 → 주소 인식 | Kakao 역지오코딩으로 클릭 위치의 주소 자동 추출 |
| 🚇 인프라 수집 | 반경 500m 내 지하철·학교·공원·병원 자동 집계 |
| 🏘️ 용도지역 분석 | 서울 UPIS SHP 파일 기반 Point-in-Polygon 판별 |
| 💰 공시지가 조회 | 법정동 단위 평균 공시지가 산출 |
| 🤖 AI 추천 | Gemini 2.5 Flash 기반 토지이용 계획 추천 |
| 📊 부지 점수 | 레이더 차트로 5개 지표 시각화 |
| 📄 PDF 저장 | 브라우저 프린트 방식으로 분석 결과 저장 |
| 🗼 서울 외 차단 | 서울 경계 외 클릭 자동 차단 |
| 💹 투자 수익성 분석 | 전월세 실거래가 기반 NOI·Cap Rate 추정 |
| 🏗️ 평균 건축연도 | 건축물대장 기반 노후도 분석 |
| 📊 권역 Cap Rate 비교 | 동일 구 내 동별 수익률 비교 테이블 |

## 기술 스택
 
**Backend**
- Python / Flask (port 5002)
- SQLite
- pyshp + shapely + pyproj (공간 데이터 처리)
- Google Gemini 2.5 Flash API
**Frontend**
- Leaflet.js / OpenStreetMap
- Stadia Maps (다크 맵 타일)
- Chart.js (레이더 차트)
- marked.js (마크다운 렌더링)
- turf.js (서울 경계 판별)
**외부 API**
- Kakao Local API (역지오코딩)

## ⚠️ 이 툴이 담지 못하는 것

**데이터 범위의 한계**
- 아파트 실거래가 기반이라 단독·다가구·상업용 부동산은 미지원
- 서울 외 지역 데이터 없음
- 데이터 주기적 갱신 로직 미구현 (현재 2025.05~2026.05 기준)

**수익성 추정의 한계**
- Cap Rate는 동별 평균 기준 추정값으로 개별 물건과 괴리 있음
- 운영비 22% 일괄 적용 (건물 유형·연식별 차등 미반영)
- 공실률 15% 고정 가정

**정성적 맥락의 한계**  
숫자로 환원되지 않는 것들이 있습니다. 이 동네가 어떤 역사를 가지고 있는지, 어떤 커뮤니티가 살아왔는지, 어떤 이유로 지금의 모습이 됐는지. 그것까지 포함했을 때 비로소 개발 방향을 말할 수 있습니다.

이를 보완하기 위해 Gemini에 주소 기반 동네 맥락을 주입해 정성적 분석이 함께 출력되도록 설계했습니다. 다만 AI가 지역의 역사와 정체성을 얼마나 정확히 읽어내는지는 여전히 열린 질문입니다.

## 실행 방법
 
### 1. 환경 설정
 
```bash
git clone https://github.com/hyeonseo801/gen-city.git
cd gen-city
pip install -r requirements.txt
```
 
### 2. API 키 설정
 
`config.py` 파일을 생성하고 아래 내용을 입력합니다.
 
```python
KAKAO_API_KEY = "your_kakao_api_key"
GEMINI_API_KEY = "your_gemini_api_key"
```
 
### 3. DB 초기화 및 데이터 로드
 
```bash
python init_db.py
```
 
### 4. 서버 실행
 
```bash
python app.py
```
 
브라우저에서 `http://localhost:5002` 접속
